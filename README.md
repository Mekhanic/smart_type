# SmartType

SmartType — контекстный автокорректор русского и английского текста для Linux.
Он работает как метод ввода Fcitx 5: исправляет опечатки и неверную раскладку,
показывает кандидатов возле курсора и умеет вернуть последнее исправление
клавишей Backspace.

SmartType развивается как ранний релиз для перечисленных ниже окружений.

## Поддерживаемые окружения

| Окружение | Статус |
|---|---|
| Fedora 44, KDE Plasma, Wayland | проверено на реальной рабочей сессии |
| Fedora 44, GNOME, Wayland | поддерживается установщиком; релизная проверка выполняется в CI |
| Ubuntu 26.04, KDE Plasma, Wayland | проверено в отдельной ВМ и после перезагрузки |
| Ubuntu 26.04, GNOME, Wayland | проверено в Writer и GNOME Terminal после чистой установки и перезагрузки; ниже указано ограничение Firefox Snap |
| Kali Rolling, Xfce, X11 | проверено в Mousepad, Firefox ESR, Kate и QTerminal после чистой установки и перезагрузки |

Поддержка относится к указанным окружениям, а не ко всем выпускам дистрибутива.
XWayland, Ubuntu 24.04 и Arch пока не являются подтверждёнными целями релиза.
Flatpak и AppImage не подходят: аддон должен загружаться процессом
Fcitx на хосте.

## Установка одной командой

Откройте обычный терминал внутри рабочего стола и выполните:

```bash
curl -fsSL https://raw.githubusercontent.com/Mekhanic/smart_type/main/scripts/install-release.sh | bash
```

Команда сама определит поддерживаемую систему и графическую сессию, скачает
готовую сборку, проверит SHA-256, установит необходимые пакеты и настроит
Fcitx. Компилятор и клонирование репозитория не нужны. `sudo` может запросить
пароль только для установки системных зависимостей.

После завершения один раз выйдите из учётной записи и войдите снова — простого
закрытия терминала недостаточно. SmartType запустится автоматически; методы
`SmartType English` и `SmartType Русский` уже будут добавлены.

Проверить установку можно одной командой:

```bash
~/.local/share/smarttype/doctor.sh
```

В выводе не должно быть строк `FAIL`.

Если вы предпочитаете сначала прочитать установщик:

```bash
curl -fLO https://raw.githubusercontent.com/Mekhanic/smart_type/main/scripts/install-release.sh
less install-release.sh
bash install-release.sh
```

Новая установка сразу создаётся во включённом состоянии. При обновлении
осознанно выключенный пользователем SmartType не включается самовольно;
вернуть его можно через меню трея или командой
`~/.local/bin/smarttypectl set-setting enabled 1`.

Для полностью неинтерактивной установки пакетов:

```bash
curl -fsSL https://raw.githubusercontent.com/Mekhanic/smart_type/main/scripts/install-release.sh | bash -s -- --yes
```

Подробности и ручной режим: [docs/INSTALL.md](docs/INSTALL.md).

## Быстрая проверка

Откройте Kate, GNOME Text Editor или другое обычное поле ввода и попробуйте:

- `севодня ` → `сегодня `;
- `руддщ ` в русской раскладке → `hello `;
- введите начало слова и выберите вариант в панели кандидатов;
- сразу после исправления нажмите Backspace — исходный текст должен вернуться.

На X11 и GNOME набираемый текст остаётся в поле приложения; панель показывает
только кандидатов. В GNOME панель выводит расширение Kimpanel, которое
установщик ставит из закреплённой и проверенной версии исходников.

Исключение: штатный Ubuntu Firefox Snap использует встроенный legacy-модуль
`fcitx4` и запрашивает клиентскую панель. Исправления SmartType в поле работают,
но расположением этой панели управляет сам Firefox, и она может оказаться не у
каретки. Это ограничение Firefox Snap, а не заявленная поддержка XWayland.

## Возможности

- автокоррекция и подсказки без отправки текста в облако;
- исправление неверной раскладки RU/EN;
- личный словарь, пользовательские правила и обучение;
- защита URL, email, путей, IP-адресов и технических токенов;
- отключение в паролях, терминалах и редакторах кода;
- история исправлений и Markdown-отчёт о проблеме из меню трея;
- единая иконка приложения и адаптивный логотип трея для светлой/тёмной темы.

Личная база находится в `~/.local/share/smarttype/personal.sqlite3`. В журнале
решений сохраняются действие, причина и хеш токена, но не полный набранный
текст, пароли или буфер обмена.

## Управление

```bash
smarttypectl add-word Happ
smarttypectl add-rule вопщем "в общем"
smarttypectl status
```

Настройки, история, словарь и отчёт о проблеме доступны через значок SmartType
в системном трее.

## Удаление

```bash
~/.local/share/smarttype/uninstall-user.sh
```

Личная база по умолчанию сохраняется.

## Разработка

Для принудительной сборки и тестирования из исходников:

```bash
git clone https://github.com/Mekhanic/smart_type.git
cd smart_type
./install.sh --build-from-source
```

Рекомендуется 4 ГБ ОЗУ, два ядра и около 5 ГБ свободного места. Обычная
установка готового релиза этих ресурсов не требует.

```bash
cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Debug
cmake --build build
ctest --test-dir build --output-on-failure
```

Архитектура и границы поддержки описаны в
[docs/ARCHITECTURE.md](docs/ARCHITECTURE.md), ручная матрица — в
[docs/DISTRO_TEST_PLAN.md](docs/DISTRO_TEST_PLAN.md).
