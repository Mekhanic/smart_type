# Security policy

## Reporting a vulnerability

Please do not publish exploitable security details in a regular issue. Use
GitHub's private vulnerability reporting for this repository:

https://github.com/Mekhanic/smart_type/security/advisories/new

Include the affected version, environment, reproduction steps, and expected
impact. Do not attach passwords, tokens, private messages, personal databases,
or other sensitive user data.

SmartType processes input locally and does not require a cloud service. A
security report is especially useful if it concerns password-field handling,
unexpected text capture, command execution, file permissions, or unsafe
installation behavior.
